# interface/

Доступ к данным по образцу MedFlow (`interface/Abstract` → `InMemory` / `sqlite`):
сервисы из [`../services`](../services/README.md) работают только с абстрактными
интерфейсами, реализация подменяется без правок бизнес-логики.

| Папка | Что внутри |
|---|---|
| `Abstract/` | Контракты хранилищ: `IUsers`, `ISessions` (дальше — `IPersonalRooms`, …). Метод `save(Model&)` присваивает `id`, если он `-1`. |
| `InMemory/` | Реализации на `QHash` — данные живут до перезапуска сервера. Используются сейчас. |
| `sqlite/` | Появится при подключении БД (Qt `QtSql`, драйвер `QSQLITE`): по классу на сущность + общий `SqliteDb` (открытие файла, схема, миграции). Весь SQL будет жить здесь. |

Модели, с которыми работают интерфейсы, лежат в [`../models`](../models/README.md).

## Правила слоя

- Интерфейс описывает **что** храним и **как ищем** (например, `findByLogin` —
  под индекс в SQLite), но не **где**: ни SQL, ни файлов, ни глобалов.
- Реализации не содержат бизнес-логики: валидация, хеширование паролей,
  правила доступа — это [`../services`](../services/README.md).
- Экземпляры создаются в `main.cpp` и передаются сервисам через
  `std::shared_ptr` (как в MedFlow).
