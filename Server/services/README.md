# services/

Бизнес-логика. Пока папка пустая — это заготовленное место: новые фичи
(регистрация, профили, история чата) реализуются здесь, а не внутри
сетевых классов.

## Правила слоя

- Сервис **не знает** про HTTP и WebSocket: принимает обычные значения или
  модели, возвращает результат или ошибку. Разбор запросов и упаковка
  ответов — забота [`../network`](../network/README.md).
- Доступ к данным — только через [`../storage`](../storage/README.md),
  никакого SQL внутри сервисов.
- Зависимости сервис получает через конструктор (реестр комнат, репозиторий
  БД), сам ничего глобального не создаёт. Точка сборки — `main.cpp`.

## Планируемые сервисы

| Сервис | За что будет отвечать |
|---|---|
| `UserService` | регистрация и вход, профиль, смена аватара, статус |
| `ConferenceService` | создание / удаление / переименование конференций, одноразовые коды приглашений, история конференций |
| `ChatService` | хранение сообщений, поиск, закрепление, реакции |
