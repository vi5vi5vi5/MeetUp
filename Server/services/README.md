# services/

Бизнес-логика. Новые фичи (профили, личные конференции, история чата)
реализуются здесь, а не внутри сетевых классов.

## Правила слоя

- Сервис **не знает** про HTTP и WebSocket: принимает обычные значения или
  модели, возвращает результат или машинный код ошибки. Разбор запросов и
  упаковка ответов — забота [`../network`](../network/README.md).
- Доступ к данным — только через интерфейсы из
  [`../interface`](../interface/README.md), никакого SQL внутри сервисов.
- Зависимости сервис получает через конструктор (`std::shared_ptr` на
  репозитории), сам ничего глобального не создаёт. Точка сборки — `main.cpp`.

## Сервисы

| Сервис | За что отвечает | Статус |
|---|---|---|
| `AuthService` | регистрация и вход, PBKDF2-хеширование паролей, сессии (выдача, проверка, скользящее продление, чистка), смена отображаемого имени | ✅ работает |
| `UserService` | аватарка, остальной профиль | план |
| `ConferenceService` | личные конференции: свой код, название, пароль, вход владельца/гостей | план |
| `ChatService` | хранение сообщений, поиск, закрепление, реакции | план |

Ошибки сервисы возвращают машинными кодами (`"login_taken"`,
`"wrong_credentials"`, …): `network/HttpApi` переводит их в HTTP-статусы,
веб-клиент — в человеческий текст. Так текст ошибок живёт в одном месте — на
странице, рядом с остальным UI.
