# core/

Модель времени выполнения сервера: кто подключён и в какой комнате.
Этот слой не слушает порты и не разбирает протокол — этим занимается
[`../network`](../network/README.md).

| Файл | За что отвечает |
|---|---|
| `ClientSession.*` | Одно WebSocket-соединение: id, имя, ссылка на текущую комнату. События сокета транслирует наверх сигналами, добавляя указатель на себя. |
| `ConferenceRoom.*` | Комната: список участников + рассылка JSON/бинарных сообщений (с возможностью исключить отправителя). |
| `RoomRegistry.*` | Реестр комнат «код → комната». Создаёт комнаты по требованию и удаляет опустевшие. |

## Владение объектами

- `ConferenceServer` владеет сессиями (`ClientSession` — QObject-ребёнок сервера);
- `RoomRegistry` владеет комнатами;
- комната хранит только «сырые» указатели на сессии и участниками **не** владеет.

Сейчас всё живёт в памяти — при перезапуске сервера комнаты пропадают.
Постоянное хранение (пользователи, история чата) появится в
[`../storage`](../storage/README.md).
