# proxy/

nginx-прокси, который делает приложение доступным по **https** — без этого
браузеры не дают доступ к камере с других устройств (`getUserMedia` работает
только в secure context: https или localhost).

| Файл | Назначение |
|---|---|
| `nginx.conf` | Маршрутизация: `/` → статика приложения, `/ws` → WebSocket-relay (с Upgrade-заголовками). TLS на 443, редирект с 80. |
| `10-gen-cert.sh` | Генерация самоподписанного сертификата при первом старте (домена нет — Let's Encrypt недоступен). |
| `Dockerfile` | `nginx:alpine` + openssl + конфиг. |

## Почему один порт на всё

Исключение для самоподписанного сертификата браузер принимает на конкретный
`host:port`. Приняв предупреждение для `https://ip/`, пользователь **не**
принял его для `wss://ip:9000` — такой сокет Firefox молча оборвёт. Поэтому
и статика, и WebSocket идут через 443, а по комнатам их разводит nginx
по пути `/ws`.

## Заметки

- Сертификат живёт в docker-томе `certs` и переживает перезапуски.
  Сгенерировать заново: `docker compose down -v && docker compose up -d`.
- nginx резолвит имя `app` при старте. Если пересоздать только контейнер
  приложения и соединения отвалятся — `docker compose restart proxy`.
- Появится домен — самоподписанный сертификат меняется на Let's Encrypt
  (certbot или caddy), остальная схема не трогается.
