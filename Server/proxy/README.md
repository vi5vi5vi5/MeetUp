# proxy/

nginx-прокси, который делает приложение доступным по **https** — без этого
браузеры не дают доступ к камере с других устройств (`getUserMedia` работает
только в secure context: https или localhost).

| Файл | Назначение |
|---|---|
| `nginx.conf` | Маршрутизация: `/` → статика приложения, `/ws` → WebSocket-relay (с Upgrade-заголовками). TLS на 443, редирект с 80, `/.well-known/acme-challenge/` для Let's Encrypt. |
| `10-gen-cert.sh` | Bootstrap: самоподписанный сертификат + симлинки `live/` (всегда, чтобы nginx стартовал). |
| `20-certbot.sh` | Получение сертификата Let's Encrypt по HTTP-01 (только если задан `DOMAIN`). |
| `30-renew-loop.sh` | Фоновое автопродление Let's Encrypt (только если задан `DOMAIN`). |
| `Dockerfile` | `nginx:alpine` + openssl + certbot + конфиг. |

## Два режима TLS

Режим выбирается переменной окружения `DOMAIN`:

- **Без домена** (`DOMAIN` не задан) — самоподписанный сертификат, как раньше.
  `10-gen-cert.sh` генерирует его при первом старте, браузер один раз предупредит.
- **С доменом** — Let's Encrypt по HTTP-01 с автопродлением:
  ```bash
  DOMAIN=meetup.linkpc.net LETSENCRYPT_EMAIL=you@mail.com docker compose up -d --build
  ```

Как это устроено: nginx.conf ссылается на стабильные пути `certs/live/{fullchain,privkey}.pem`
(симлинки). До старта nginx `10-gen-cert.sh` направляет их на самоподписанный сертификат —
nginx стартует гарантированно. Если задан `DOMAIN`, `20-certbot.sh` в фоне дожидается старта
nginx, получает сертификат Let's Encrypt через webroot и переставляет `live/` на него
(`nginx -s reload`). Если запрос не удался (нет DNS/сети) — остаёмся на самоподписанном,
сервер работает.

## Почему один порт на всё

Исключение для самоподписанного сертификата браузер принимает на конкретный
`host:port`. Приняв предупреждение для `https://ip/`, пользователь **не**
принял его для `wss://ip:9000` — такой сокет Firefox молча оборвёт. Поэтому
и статика, и WebSocket идут через 443, а по комнатам их разводит nginx
по пути `/ws`.

## Заметки

- Самоподписанный сертификат живёт в томе `certs`, сертификаты Let's Encrypt —
  в томе `letsencrypt`. Оба переживают перезапуски.
  Сбросить всё (перевыпуск с нуля): `docker compose down -v && docker compose up -d`.
- nginx резолвит имя `app` при старте. Если пересоздать только контейнер
  приложения и соединения отвалятся — `docker compose restart proxy`.
- **HTTP-01 всегда идёт на внешний порт 80**, поэтому в режиме с доменом
  `HTTP_PORT` переопределять нельзя — контейнер должен быть опубликован на `:80`.
- **Rate limit Let's Encrypt** на боевые сертификаты жёсткий: том `letsencrypt`
  не удалять без нужды, а для отладки добавлять флаг `--staging` в `certbot`
  внутри `20-certbot.sh` (staging-сертификат браузер не примет, зато лимиты щадящие).
- Проверить автопродление вручную: `docker compose exec proxy certbot renew --dry-run`.
